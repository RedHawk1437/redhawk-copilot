# RedHawk Copilot Project Rules

---

## Learning Philosophy

1. Learning comes before speed.
2. Never copy code without understanding it.
3. Every function must include comments explaining its purpose.
4. Start simple, improve gradually.
5. Follow Python best practices.
6. Follow ethical hacking principles only.
7. Work only in authorized environments.
8. Every feature must solve a real problem.
9. Every feature must teach something new.
10. Keep documentation updated.
11. Every coding session ends with:
    - Testing
    - DevLog update
    - Architecture update (if needed)
    - Learning Journal update
    - Mission Log update (if needed)
    - Project Context update
    - Git Commit
    - Git Push
12. Important decisions must never remain only in chat.
13. Use real lab data whenever possible.
14. Understand first, code second, optimize later.
15. AI-generated code is accepted only after understanding every line.
16. If documentation and code disagree, update the documentation immediately or fix the code. Never let them drift apart.
